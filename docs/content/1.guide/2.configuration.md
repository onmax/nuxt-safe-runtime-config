All options for `safeRuntimeConfig` in your `nuxt.config.ts`.

## Options Reference

| Option              | Type                            | Default           | Description                                        |
| ------------------- | ------------------------------- | ----------------- | -------------------------------------------------- |
| `$schema`           | `StandardSchemaV1 \| string`    | -                 | Inline schema or default-exported schema module    |
| `validateAtBuild`   | `boolean`                       | `true`            | Validate during development and build              |
| `validateAtRuntime` | `boolean`                       | `false`           | Validate after Nitro applies runtime overrides     |
| `onError`           | `'throw' \| 'warn' \| 'ignore'` | `'throw'`         | Control validation error behavior                  |
| `jsonSchemaTarget`  | `string`                        | `'draft-2020-12'` | Select the JSON Schema draft for inline validation |

Set `$schema` to a project-relative module path when the schema transforms runtime input. The module must default-export a Standard Schema implementation.

## Example Configuration

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  safeRuntimeConfig: {
    $schema: runtimeConfigSchema,

    // Validation timing
    validateAtBuild: true, // default
    validateAtRuntime: false, // opt-in

    // Error handling
    onError: 'throw', // 'throw' | 'warn' | 'ignore'

    // JSON Schema
    jsonSchemaTarget: 'draft-2020-12',
  },
})
```

## Error Behaviors

### `'throw'` (default)

Stops the build or throws an error at runtime. Recommended for catching issues early.

### `'warn'`

Logs a warning but continues. Useful during migration or when some values are expected to be missing in development.

### `'ignore'`

Silently continues. Use sparingly.

## JSON Schema Target

The `jsonSchemaTarget` option controls which JSON Schema draft inline schemas use for runtime validation. Schema modules execute the original Standard Schema and ignore this option.

Available targets include:

- `'draft-2020-12'` (default) - latest, best compatibility
- `'draft-2019-09'` - if your tools require it
- `'draft-07'` - legacy support

Inline runtime schemas use [@cfworker/json-schema](https://github.com/cfworker/cfworker/tree/main/packages/json-schema), a lightweight validator that works on edge runtimes. A string `$schema` path executes the original Standard Schema instead, so runtime transformations are preserved.

When a schema library does not expose native JSON Schema output, the module falls back to [@standard-community/standard-json](https://github.com/standard-community/standard-json). Valibot and Zod converter dependencies are bundled with the module package, so application projects do not need extra installs for those libraries.
